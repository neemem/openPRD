# v4.4.6 - 2026-06-08 - Step 0 自适应重构 + G0 门控 + 轻量模式

## 一句话总结

主 Agent 在 Step 0 改为"先静默自动判断 7 维度,只问判不准的(通常 1-4 问)",新增 G0 复述门控、轻量 3 路调度、6 题二期版,把"用户答完 6-8 问才开始"压到"通常 1-2 问 → G0 复述 → 启动",同时清理 12 个文档残留(v4.4.5 删除 architecture-mindmap 后的残留 + 子技能计数错误)。

---

## 背景与动机

用户反馈:
1. **"现在整个技能流程上你再检查一下,有没有什么可能我没注意的地方"**——隐性需求是审计
2. **"在与用户确认问题时,是不是考虑给更多的有价值的选项,还有给到更多的问题"**——隐性需求是问题质量
3. **"我觉得多少问不重要,主要为了产出质量,都可以,但是要考虑到是什么样的应用,要更自动化更精准地提问"**——核心约束:**自动 > 多问**

**矛盾**:用户既想要"更多有价值的问题",又想要"更自动化少打扰"。**解法**:主 Agent 先自动判 7 维度,只问判不准的,让用户感受"被理解",而不是"被审问"。

#### 4. Step 0 整体重构(自适应 + 合并)
- **位置**:`SKILL.md` lines 33-138
- **症状**:v4.4.4 的 Step 0 问 6-8 个互相关联的问题(后端/前端、页面/功能、是否 BI、产出范围、模式...),用户答得累,主 Agent 没"自动判断"能力,全部依赖用户回答
- **修正**:
  - 新增"自动判断优先级":主 Agent 静默扫描用户原始消息,提取 7 维度(主题/用户/项目类型/前后端/页面级/BI/合规)

  - 4 个问题合并(从 6-8 问压到 1-4 问):
    - **A 需求来源**:3 选(我是需求方 / 转答他人 / 待调研)
    - **B 交付范围 9 选 1**:把"后端/前端" × "页面/功能" × "是否 BI" 三链合并为 9 选 1(强相关决策链合并)
    - **C 产出范围 4 类多选**:产品 / 技术 / 运营 / 合规(每类对应固定文档集)
    - **D 行业合规 + 时间**:仅强合规项目才问,时间分 4 档(1-3 天 demo / 1-2 周 MVP / 1-3 月 / 3 月+)
  - 每问带"为什么"+"我不确定"兜底
  - 5 类互斥检测(页面级+简化+仅前端 / BI+简化+功能级 / 仅技术+含产品文档 / 排除合规+金融医疗 / 海外+排除合规)
- **影响**:平均问题数 -50%(6-8 → 1-4),用户决策疲劳大幅下降

## 用户使用变化

**老流程(v4.4.4)**:
```
用户: "做一个内部审批工作流"
Agent: "请回答 6 个问题:1. 后端还是前端? 2. 页面还是功能? 3. 要 BI 吗? 4. 给谁看? 5. 时间? 6. 行业?"
用户: 答 6 个,5 分钟
Agent: 开始 35 题
```

**新流程(v4.4.6)**:
```
用户: "做一个内部审批工作流"
Agent: 输出画像卡片(7 维度自动判)+ 仅问 1-3 个判不准的
用户: 答 1-3 个,1 分钟
Agent: G0 复述 → 用户"OK" → 35 题
```

**节约**:平均 4 分钟决策疲劳 + 大幅降低"被审问感"。
