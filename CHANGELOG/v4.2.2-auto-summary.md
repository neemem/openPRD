# v4.2.2 - 2026-06-06 - 自动化摘要填充

## Context

v4.2.1 承诺了"被排除受众对应文档降级为 ≤ 200 字摘要",但 reference/19 §3.2 算法只是 Markdown 中的 Python 伪代码,**不可执行**——5 个 v4.3 新模板末尾的"## 摘要"段都是 `> ⚠️ 待 v4.2.2 填充` 占位。v4.2.2 落地算法为可执行 Python 脚本,集成到 `openPRD-document-generation` Step 3 调度。

## 新增

| 项 | 路径 | 价值 |
|----|------|------|
| **200 字摘要提取脚本** | `reference/19-summarizer.py`(可执行,纯 Python 3.6+ 标准库) | 实现 reference/19 §3.2 算法,零外部依赖;支持 stdin/文件输入,3 个测试用例全过 |
| **openPRD-document-generation 受众感知调度** | `openPRD-document-generation/SKILL.md` "### 受众感知调度(v4.2.2)"章节 | Step 3 每个 Track 完成后、文件写入前执行;主受众不在 `default_audiences` → 调用 summarizer 生成摘要 append 到"## 摘要"段 |
