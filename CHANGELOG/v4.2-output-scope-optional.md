# v4.2 - 2026-06-06 - 产出范围可选化

## Context

v4.1 之前,openPRD 默认全产出 16 份文档 + 5 份行业研究,但实际项目中不同角色(PM / 后端 / 法务 / 运营)关心点差异巨大——产品经理不需要 03-接口文档的完整 DDL,后端工程师也不需要 14-行业分析的 9 章节。v4.2 引入"产出范围可选化":用户在 Step 0 第四问选择 4 类粗粒度受众(产品/技术/运营/合规),被排除的受众对应文档**降级为 ≤ 200 字摘要**(核心决策 bullet + 关键数字/对象 + 完整版链接)。**默认全选**,完全兼容 v4.1 老用户。**特殊规则**:合规章节即使被排除,核心 5 条(GDPR/个保法/等保/行业资质/跨境传输)仍保留为摘要中的 5 个 bullet——合规风险零容忍。

## 子技能 Gotchas(产出范围相关,共 5 条)

详见 `reference/19-audience-tag-table.md` "常见误用"章节(§8),5 条覆盖:

- A-01: 用户说"我只关心后端",`default_audiences = ["技术"]` → 产品文档全降级,后端无法理解"做什么、为什么" → 至少保留"产品"+"技术"
- A-02: 合规章节被完全排除(连核心 5 条都没有) → 遗漏 GDPR / 个保法关键条款 → §3.5 特殊规则强制保留核心 5 条
- A-03: 摘要超 200 字 → 失去"降级"意义 → 严格按 §3.1 固定结构 + §3.2 截断算法
- A-04: 摘要只有"模板描述"没有"核心决策" → 摘要成了"目录" → 必须含 3-5 个 ⭐ 决策 bullet
- A-05: 误用 `default_audiences` 做"权限控制" → 用户看不到完整内容 → 摘要阶段所有受众均可访问完整文档链接(只是默认不展开)
