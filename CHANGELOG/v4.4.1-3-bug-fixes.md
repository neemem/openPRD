# v4.4.1 - 2026-06-08 - 架构脑图 v4.4.0 三处问题修复

## Context

v4.4.0 在 wudi 项目首次试跑后,用户反馈 3 处问题:
1. **XMind 文件损坏** — XMind 2020+ 报 "this file may have become corrupted",因缺 `META-INF/manifest.xml` + `creator` 字段位置错误
2. **产品脑图过于简单** — L4 页面被折叠为 `(N 项)` 注解,22 个功能点的页面信息密度低
3. **没有关系链** — 2 张脑图都只是层级结构,缺少跨节点关联(数据流/跳转/鉴权/共享)

## 修复

| # | 问题 | 修复 | 文件 |
|---|------|------|------|
| 1 | XMind 损坏 | + `META-INF/manifest.xml`(2020+ 必备);`creator` 移至 `content.json` 顶层;节点附加数据改用 `notes.plain.content`(XMind 标准字段) | `reference/20-mindmap-generator.py:289-348` |
| 2 | 产品脑图简单 | Mermaid/Markdown 渲染时,如节点有 `pages[]`,展开为可见 L4 子节点(共 26 个 L4 页面) | `reference/20-mindmap-generator.py:render_mermaid` |
| 3 | 无关系链 | 新增 `relationships[]` 数组(YAML/JSON),4 格式均渲染:Mermaid 独立 `flowchart LR` 块、Markdown "## 跨节点关系链" 表格、JSON `relationships` 数组、XMind `sheet.relationships` | `reference/20-mindmap-generator.py:render_*` |

## 关系链类型

| 类型 | 用途 | wudi 产品脑图(34 条) |
|------|------|---------------------|
| `data_flow` | 数据流 (T-1 聚合→消费端) | 11 |
| `authority` | 鉴权依赖 (auth→所有页面) | 15 |
| `navigation` | 导航跳转 | 2 |
| `event` | 事件触发 | 3 |
| `shared_entity` | 共享实体 | 3 |
