## 4 轮迭代失败根因(供下次避坑)

| 迭代 | 用户反馈 | 修复 | 用户是否接受 |
|------|----------|------|------------|
| v4.4.0 | (初版) | 双脑图 × 4 格式 | 部分 |
| v4.4.1 | XMind 打不开 / 产品过简 / 缺关系 | 3 处修复 | 部分 |
| v4.4.2 | XMind 还是不能打开 | XMind 25.x 5 处致命错误 | 部分 |
| v4.4.3 | 连线太多 / L2 描述少 | priority + description 字段 | 部分 |
| v4.4.4 | 连线还是过多 | 强降到 5 + 视觉降密度 | **拒绝(无法落地)** |

**根本问题**(反思):
1. **技能定位不清**:可视化层 vs 完整关系图未分清,用户期望与设计目标错位
2. **缺真实用户试用**:只做"自检报告"(技术正确),没做"用户 5 分钟演示"(真实可用)
3. **应暂停加功能**:2 轮迭代后用户仍不满意,应转做用户调研,而非继续加 priority/description/primary_target 等配置项
4. **失去用户信任**:3 轮反复后,即使 v4.4.4 设计正确(5 分钟法则 + 视觉降密度),用户也已失去信任,删除比保留更好
