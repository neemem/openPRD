# v4.4.0 - 2026-06-08 - 架构脑图生成(业务+产品 双脑图 × 4 格式)

## Context

用户提出"产品架构脑图,作为可展示资产,支持 xmind"需求。openPRD 此前只能产出 16+ 份"文档型"资产(PRD/接口/架构等),缺少"可视化展示型"资产。本次升级新增 1 个子技能 + 1 个生成器 + 1 份模板,填补这一空白。

**关键设计**:
- **2 张脑图**:业务架构脑图(老板/客户视角)+ 产品架构脑图(开发/测试视角),覆盖 2 个受众
- **4 种格式**:Mermaid mindmap(嵌入文档)+ Markdown 大纲(可读性)+ JSON 树(程序消费)+ XMind 文件(原生 XMind 打开)
- **零外部依赖**:XMind 文件用 stdlib `zipfile + json` 写,无需安装 `xmind` Python 库
- **8 份文件**:1 次调用产出 2 × 4 = 8 份可视化资产

## 关键决策样本(⭐)

- **⭐ 决策 1 (AM3.1)**:脑图深度硬上限 5 层,超出折叠为 "… (N 项)"。原因:Mermaid mindmap 的 SVG 定位算法深度 > 5 失稳,XMind 自动折叠深层节点。
- **⭐ 决策 2 (AM3.2)**:每层兄弟节点 3-7 个。超 9 必拆层。原因:Miller's Law 工作记忆 7±2,超 9 节点读者无法整体把握。
- **⭐ 决策 3 (AM6.1)**:业务/产品 2 张脑图默认都出。原因:用户原始需求"出两张"。
- **⭐ 决策 4 (AM6.3)**:接口进产品脑图 L5。原因:开发/测试看脑图就知道"每个页面对应哪些 API",省一次查 03 的时间。
- **⭐ 决策 5 (G-08)**:产品脑图 L2 必含"管理后台"。原因:完整性是产品脑图的最低标准,运营/客服/老板不可见 = 产品不完整。
- **⭐ 决策 6 (G-10)**:业务架构 = 业务域(非组织部门)。原因:业务域是业务能力的自然聚合,跨组织架构稳定。
