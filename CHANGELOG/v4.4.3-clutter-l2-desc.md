## 关键改动

### 1. 关系加 `priority` 字段(主链路 vs 横切)

```yaml
# v4.4.3 新增字段
relationships:
  - source: record_contribute
    target: sales_record_cap
    label: "录入即沉淀"
    type: data_flow
    priority: primary       # ← 新增(进入 XMind/Mermaid 可视化)
  - source: auth_rbac
    target: audit_log
    label: "操作→留痕"
    type: cross_cutting
    priority: secondary     # ← 新增(仅入 Markdown 附录 + JSON)
```

- **未指定 priority 视为 primary**(向后兼容旧 YAML)
- 业务 18 条 → 10 primary + 8 secondary
- 产品 34 条 → 10 primary + 24 secondary

### 2. 4 格式分层渲染

| 格式 | 显示关系 | 说明 |
|------|----------|------|
| **Mermaid** | 10/10 primary | flowchart LR 块只画主链路,降 clutter |
| **XMind** | 10/10 primary | sheet.relationships 数组只含 primary |
| **Markdown** | 10 + 8 / 10 + 24 | 拆为两张表:"主链路" + "附录:横切/次要关系" |
| **JSON** | 18/34 全量 | 保留全部,含 priority 字段供程序消费 |

### 3. L2 节点加 `description` 字段

```yaml
- id: market_data_domain
  name: 行情数据域
  description: "兰花品种交易行情的采集、聚合、计算与展示核心域,..."
  children: [...]
```

- 业务脑图 5 个 L2 全部有描述
- 产品脑图 5 个 L2 全部有描述
- 显示位置:Markdown "说明" 列表项 + XMind notes(以 📌 开头) + JSON 字段

### 4. Mermaid 关系按类型配色

```mermaid
classDef dataFlow stroke:#1F6FEB,stroke-width:2px;     # 蓝-数据流
classDef eventFlow stroke:#DC2626,stroke-width:2px,stroke-dasharray:4;  # 红-事件
classDef navFlow stroke:#059669,stroke-width:2px;      # 绿-导航
```
