## V5.0 核心改进(14 项)

### P0 必修(4 项)

#### P0-001:文件大小自动验证(`min_size_per_doc`)

- **背景**:V4.4 没有最小文件大小约束,V1 初版 39 份文档平均仅 10KB
- **方案**:在 `.openprd/config.json` 新增字段
  ```yaml
  min_size_per_doc:
    P0_core: 40000       # PRD/接口/DB/架构
    P1_important: 30000   # 开发指南/设计/交互
    P2_brief: 20000       # README/汇总
  ```
- **实施位置**:Subagent 完成时自检 + 主 Agent 验收复检
- **影响**:从 V4.4 平均 10KB → V5.0 平均 47KB(+370%)

#### P0-002:文档质量分级

- **方案**:19 个文档明确分级 + 最低/推荐字节
- **示例**:06-产品需求文档.md = P0(40K/60K),01-README.md = P2(20K/40K)

#### P0-003:Track A 并行化

- **背景**:V4.4 主 Agent 串行写 13 份 PRD ~10min
- **方案**:拆给 4 个 Subagent 并行(~3-4min,节省 60%)
- **新分工**:A1-主(06 主文档)/ A2-拆分-上(06a/06b/06c)/ A2-拆分-下(06d/06e/06f)/ A3-发布+附录(06g/06h/06i)

#### P0-004:V1.5 扩充阶段显式化

- **背景**:用户事后反馈"内容太少"
- **方案**:
  ```
  V1 (75min) → V1.5 Phase(扩充,~10min) → V2(交付,~5min)
  ```
- **触发条件**:用户输入含"详细/深入/40K" / 文档平均 < 30KB / 项目复杂度=高

### 置信度内容考量(本次新增)

**5 维度 100 分制评分卡**:

| 类别 | 分值 | 维度 |
|---|---|---|
| **A. 引用量** | 40 分 | 章节完整度/表格密度/Mermaid/代码 |
| **B. 引用依据专业性** | 35 分 | 决策整合/来源数量/来源权威性 |
| **C. 是否有背书** | 25 分 | 内部决策/行业案例/合规/实践 |

**引用来源权威性 5 级分级**:

| 等级 | 标识 | 计分 | 示例 |
|---|---|---|---|
| 🟢 **权威** | 一级 | 3 分/个 | 工信部法规、ISO 标准 |
| 🟢 **行业** | 二级 | 2 分/个 | 行业协会报告 |
| 🟡 **厂商** | 三级 | 1.5 分/个 | 阿里云官方文档 |
| 🟡 **推断** | 四级 | 0.5 分/个 | 经验推断(需标注) |
| 🔴 **未验证** | 五级 | 0 分 | 无权威来源 |

**置信度分级**:
- 90-100: A 级 ⭐⭐⭐⭐⭐ 直接交付
- 80-89: B 级 ⭐⭐⭐⭐ 交付
- 70-79: C 级 ⭐⭐⭐ 触发自动扩充
- 60-69: D 级 ⭐⭐ 重写
- <60: E 级 ⭐ 完整重写

### Subagent 失败持久化 + 断点续做(本次新增)

- **机制**:每次 Subagent 调用前保存 checkpoint → 成功后删除 → 失败后保留
- **state.json** 记录:
  - 已完成任务列表
  - 失败任务(含错误信息/重试次数)
  - 待执行任务列表
- **断点续做**:下次会话自动检测 state.json,询问用户继续/重开/查看

### 强制生成完整执行报告(本次新增)

- **每次 Skill 运行必生成**:`OpenPRD-执行报告-{YYYYMMDD}.md`
