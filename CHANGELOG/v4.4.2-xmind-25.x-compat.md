# v4.4.2 - 2026-06-08 - 架构脑图 XMind 25.x 兼容性修复

## Context

v4.4.1 后用户(XMind 25.01.01061)报告"XMind 还是不能打开"。根因分析:
将生成器与一份真实可用的 XMind 25.x 文件(`/Users/neemem/WORK/米链AI产品.xmind`)对比,发现 v4.4.1 错误地基于 XMind 2018/2020 早期规范实现,实际 XMind 25.x 用了一套完全不同的文件结构。

## 5 个关键修复

| # | 错误 | 修复 |
|---|------|------|
| 1 | `content.json` 是 object `{sheets:[...]}` | **改为 array `[{sheet1}]`** — XMind 25.x 顶层就是数组 |
| 2 | 用了 `META-INF/manifest.xml` | **改为根级 `manifest.json`** — 真实 XMind 用 `{"file-entries":{...}}` 格式 |
| 3 | ZIP 用 `ZIP_DEFLATED` 压缩 | **改为 `ZIP_STORED`(不压缩)** — 真实 XMind 文件全部 STORED |
| 4 | 关系字段 `sourceId` / `targetId` | **改为 `end1Id` / `end2Id`** — XMind 原生字段名 |
| 5 | 缺 `Thumbnails/thumbnail.png` | **加 1x1 透明 PNG 占位** — 缺则部分版本报"corrupted" |

## 附加改进

- 新增 `content.xml`(XMind 8 兼容格式),xmind Python 库和 XMind 8 可读
- Topic 加 `attributedTitle` 字段(与 title 一致),保证 XMind 渲染
- Theme 块加 `centralTopic` / `mainTopic` / `subTopic` 样式,符合 XMind 25.x 主题规范
- `metadata.json` 简化为 `{dataStructureVersion, layoutEngineVersion, creator}` 三字段

## 验证 (用 xmind Python 库)

## 文件清单 (6 文件,与真实 XMind 一致)

```
新兰界-business-mindmap.xmind
├── content.json          (array, 含 sheet + relationships)
├── content.xml           (XMind 8 兼容)
├── metadata.json         (dataStructureVersion + creator)
├── manifest.json         (根级别)
├── xmind.css             (默认样式)
└── Thumbnails/thumbnail.png  (1x1 透明 PNG 占位)
```
