# 用户调研问卷（35题完整版 · 9 组）

> 📖 **使用说明**：本问卷由 `openPRD-user-research` 在调研阶段使用，按 9 组分组，每组提问后**必须等待用户回答**才能继续。
>
> **Why 35 题**：用户反馈"几个简短问题无法确认清楚需求"。35 题覆盖业务/用户/功能/技术/数据/合规/运营/设计/收口全维度，避免任何"想当然"。

---

## 设计原则

| 原则 | 说明 |
|------|------|
| **分组归总** | 按 9 大主题分组，每组有明确目标 |
| **多维度覆盖** | 业务 + 用户 + 功能 + 技术 + 数据 + 合规 + 运营 + 设计 + 收口 |
| **核心问题先行** | Q1-Q5 关键问题先问，剩余问题在等待期间并行调研 |
| **总量控制** | **35 个问题**（含 5 个核心 + 25 个深度 + 5 个收口）|
| **简洁明确** | 每个问题目的明确，不产生歧义 |
| **必答 + 可选** | 标记 ⭐ 为必答题，其余可选（鼓励但允许跳过）|

### 问题分组结构

```
【第一组】交付范围（4 题，⭐ 核心）    ← 必须先问，等待回答
        ↓
【第二组】业务背景（5 题）            ← 等待期间并行行业调研
        ↓
【第三组】用户与场景（5 题）
        ↓
【第四组】功能需求（4 题）
        ↓
【第五组】技术约束与集成（4 题）
        ↓
【第六组】数据与合规（4 题，⭐ 合规）   ← 涉及个人信息/支付/跨境等
        ↓
【第七组】运营与增长（3 题）
        ↓
【第八组】设计偏好与体验（3 题）
        ↓
【第九组】收口确认（3 题，⭐ 必答）   ← 确认理解、补漏、决定下一步
```

## ⏱️ 调研执行规范

### 总时长预估
- 核心问题（第一组 4 题）：**5-10 分钟**
- 深度问题（第二-八组 28 题）：**30-50 分钟**（按用户回答深度调整）
- 收口确认（第九组 3 题）：**5-10 分钟**
- **总时长**：约 40-70 分钟

### 必须等待用户回答

```
Q1 → 等待 → Q2 → 等待 → Q3 → ... → 等待 → Q35 → 产出报告
```

**绝对禁止**：批量提问后让用户一次性回答，必须**一问一答**。

### 关键问题先行的"并行调研"原则

> 在用户回答**第一组核心问题（Q1-Q4）后**，**立即并行启动**：
>
> 1. **行业分析** subagent（基于 Q1+Q2 的主题和目标用户）
> 2. **竞品分析** subagent（基于 Q1+Q2 的产品类型）
> 3. **标杆研究** subagent（基于 Q1 的产品类型）
>
> 这 3 个 subagent 的产出**不需要等待** Q5-Q35 的答案，可以**基于早期需求画像**预研版本（标记置信度）。
>
> 当用户回答完 Q5-Q35 后，将答案**回填**到 subagent 产出中做修订。

### 必答 vs 可选

- ⭐ **必答**：Q1/Q2/Q3/Q4/Q5/Q6/Q9/Q10/Q11/Q15/Q16/Q17/Q20/Q25/Q26/Q31/Q33/Q35（共 18 题）
