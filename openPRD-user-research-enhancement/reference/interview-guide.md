# 访谈提纲结构

## 区块 1：Warm-up（5 分钟）— 建立信任

| # | 问题 | 目的 | 探针（Probe） |
|---|------|------|----------------|
| 1.1 | 您好，感谢您抽出时间参加这次访谈。请先简单介绍一下您自己，包括您做这项工作多久了？ | 破冰，了解背景 | - |
| 1.2 | 您目前主要负责哪些类型的任务？每天大概处理多少？ | 建立场景上下文 | "这个量级在团队里算多还是少？" |
| 1.3 | 在正式开始之前，我想说明：这次访谈没有"正确答案"，我只是想了解您真实的工作体验，您可以放心畅所欲言。 | 消除顾虑，明确目的 | - |

## 区块 2：Context（10 分钟）— 理解当前工作流

| # | 问题 | 探针（Probe） |
|---|------|--------------|
| 2.1 | 请带我走一遍您处理一个典型任务的完整流程，从接到任务到任务完成。 | "能说得更具体一点吗？" / "这一步大概花多长时间？" / "这一步您觉得顺畅吗？" |
| 2.2 | 您觉得目前工作中最耗时的环节是什么？ | "为什么觉得这个环节耗时？" / "有没有尝试过什么方法改善？" / "改善后效果如何？" |
| 2.3 | 您一般使用哪些工具或系统？它们之间如何配合？ | "哪些工具是您每天都会用到的？" / "哪个工具让您觉得特别好用或特别不好用？" / "这些工具之间数据是如何同步的？" |
| 2.4 | 您一般如何判断任务的优先级？有哪些参考标准？ | "有没有遇到过难以判断优先级的情况？" / "当多个任务同时来时，您怎么处理？" |

## 区块 3：Deep Dive（20 分钟）— 深入探索痛点与需求

| # | 主题 | 问题 | 探针 |
|---|------|------|------|
| 3.1 | 任务分配 | 任务是如何分配给您的？您能影响这个分配过程吗？ | "您理想中的分配方式是什么样的？" / "现在的分配方式有什么让您不满的？" |
| 3.2 | 信息获取 | 当您需要查看任务详情时，这个过程顺畅吗？ | "您一般要点击几次才能看到完整信息？" / "最让您困扰的是什么？" / "有没有找不到想要信息的时候？" |
| 3.3 | 协作与流转 | 当任务需要跨部门协作时，通常会发生什么？ | "协作过程中最头疼的是什么？" / "协作延迟通常是因为什么？" / "您通常需要催别人吗？" |
| 3.4 | 效率工具 | 您平时会使用快捷回复模板吗？如果没有，是因为什么原因？ | "您希望模板具备哪些功能？" / "模板多到什么程度会变得难以管理？" |
| 3.5 | 工作负荷 | 在高峰期，任务堆积的情况严重吗？您通常如何应对？ | "系统能做些什么来帮助您？" / "您有什么 workaround 吗？" |
| 3.6 | 绩效反馈 | 您平时能清楚地看到自己的工作量数据吗？ | "您希望看到哪些维度的数据？" / "现有的数据看板有什么让您不满意的？" |

## 区块 4：Reaction（10 分钟）— 概念/原型反应

| # | 活动 | 提问 | 追问 |
|---|------|------|------|
| 4.1 | 展示设计概念或原型 | "您看到这个设计后，第一反应是什么？" | "让您有这个反应的原因是什么？" |
| 4.2 | - | "您觉得这个功能对您的工作会有什么帮助？" | "具体会帮到哪个场景？" |
| 4.3 | - | "如果要您给这个功能打分（1-10），您会打几分？理由是什么？" | "如果想拿到更高的分数，还需要什么？" |
| 4.4 | - | "如果这个功能上线了，您第一个想用它来做什么？" | "为什么这件事对您最重要？" |

## 区块 5：Wrap-up（5 分钟）— 收尾与感谢

| # | 问题 |
|---|------|
| 5.1 | 今天聊了很多，有没有什么是我们没有谈到但您觉得重要的？ |
| 5.2 | 如果您可以改变现有系统的一个功能，您会改什么？为什么？ |
| 5.3 | 感谢您的参与！您有没有任何问题想问我？ |
