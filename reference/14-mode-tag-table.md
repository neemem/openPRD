# 14 · 模式标签单一来源（Mode Tag Table）

> 📖 **使用说明**：本文档是 openPRD **模式标签**（通用 / 仅完整 / 可选）的**唯一来源**。
> 模式标签决定：每份模板在「完整模式」「简化模式」「按行业需要」下是否产出。
> 引用方：`templates/00-INDEX.md` §0.1 / `reference/05-quality-gates.md` §4.1 / 主 SKILL.md 文档清单 / `openPRD-document-generation/SKILL.md`。
>
> **Why**：原体系下"通用 / 仅完整 / 可选"标签在 4+ 处独立维护（00-INDEX / 文档清单 / G3 / execution-steps），标签更新遗漏即出现"漏文档 / 多文档"。

## 0. 文档目的

- **单一来源**：一份表决定每份模板的产出范围
- **跨模式一致性**：完整模式 + 简化模式 文档交集 = 通用 标签
- **可机器消费**：下表可被脚本读取，生成 `output-manifest.json`

## 1. 交付模式定义

| 模式 | 触发条件 | 文档范围 | 典型场景 |
|------|----------|----------|----------|
| **完整模式** | 用户选 A（后端同时实现） | 通用 + 仅完整 = 16 份 | 客户交付、商业项目、复杂系统 |
| **简化模式** | 用户选 B（只考虑前端） | 通用 = 9 份（含用户调研 + 14-行业分析 共 9） | 内部产品、快迭代、Mock 优先 |

| **可选（行业）** | 涉及特定行业（医疗/金融/教育/政务） | 14-行业分析 + 14-合规清单 | 强监管场景 |

## 4. 模式标签判定规则（agent 决策依据）

### 4.1 用户输入读取

| 输入片段 | 模式 | 标签启用 |
|----------|------|----------|
| "后端同时实现" / "全栈" / "需要数据库" | 完整模式 | 通用 + 仅完整 |
| "只考虑前端" / "Mock 优先" / "前端先跑" | 简化模式 | 仅通用 |
| "页面级" / "页面清单" / "页面方案" / "出页面" | 强制 16 章 PRD 拆分 | 通用（页面级强制）|
| "行业研究" / "强监管" / "医疗/金融/政务" | 行业研究 5 路启用 | 可选标签启用 |
| 无明确指定 | 默认完整模式 | 通用 + 仅完整 |

### 4.2 模式冲突优先级

```
页面级强制 > 完整/简化模式 > 行业可选
```

- 即使用户选简化模式，触发"页面"关键词 → 强制页面级 PRD 拆分
- 即使在完整模式，未涉及强监管行业 → 14-合规清单不产
- 即使用户没选，行业研究 5 路已启用 → 14-行业分析必产

### 4.3 标签更新的影响范围

任何标签变更必须**同步更新以下 3 处**：

| 位置 | 影响 |
|------|------|
| 本文档（reference/14） | 模式标签总表 |
| `templates/00-INDEX.md` §0.1 | 模板列表"模式"列 |
| `openPRD-document-generation/reference/execution-steps.md` §1 | 产出范围对照表 |

> 单点更新会立即造成"标签漂移"，所有改动必须**先改本表，再改下游引用**。

## 5. 模式标签在 G3 完整性检查中的用法

> 详见 `reference/05-quality-gates.md` §4.1 完整性检查 + `reference/13-quality-selfcheck.md` §3.1。

主 Agent 在 G3 完整性自检时：

```bash
# 完整模式
ls <project>-docs/ | grep -E "^(01|02|03|04|05|06|07|08|09|10|11|12|13|FigmaMake-Prompt|用户调研报告|14-行业)" | wc -l
# 期望 = 14 + 14-行业（可选）+ 用户调研 = 16

# 简化模式
ls <project>-docs/ | grep -vE "(08|09|12|13|14-)" | wc -l
# 期望 = 9 + 用户调研 = 10
```

## 6. 反模式（标签错用）

| 反模式 | 后果 | 修正 |
|--------|------|------|
| 简化模式下产出 12-数据库设计 | 前端无后端却写了表结构 | 删除文档（"通用"逻辑下不产出）|
| 完整模式下漏产 09-后端开发指南 | 后端无规范，代码风格混乱 | 补 09 |
| 触发"页面"词但未拆 06a-06h | PRD 太粗，用户投诉 | 强制拆 8 段 |
| 行业研究 5 路启用却漏产 14-合规 | 强监管项目无合规检查 | 补 14-合规 |
| 标签在 00-INDEX 与本文档不一致 | agent 决策矛盾 | 以本文档为准并同步 INDEX |
