# 28 · 医疗软件(SaMD)领域扩展指南(V5.0+ 待实施)

> 📖 **本文档定位**:
> - **用户场景明确**:纯软件医疗产品(医生问诊/患者随访/慢病管理/医学 AI 辅助诊断等)

## 2. SaMD 风险分类体系(IMDRF)

### 2.1 状态分类

| 状态 | 含义 | 示例 |
|---|---|---|
| **I. 非紧急治疗** | 慢病管理、健康教育 | 心理健康 App、慢病随访 |
| **II. 紧急治疗** | 急性病监护、辅助决策 | 急救辅助、危重监护 |
| **III. 危重治疗** | 生命支持 | 体外生命支持软件 |
| **IV. 紧急/危重治疗** | 抢救场景 | ICU 监护 |

### 2.2 影响分类

| 影响 | 含义 |
|---|---|
| **a. 信息性** | 提供参考信息 |
| **b. 辅助性** | 辅助医师决策 |
| **c. 驱动/治疗** | 直接影响治疗 |

### 2.3 风险组合 → 监管要求

| 风险等级 | I | II | III | IV |
|---|---|---|---|---|
| a(信息) | I-a | II-a | III-a | IV-a |
| b(辅助) | I-b | II-b | III-b | IV-b |
| c(驱动) | I-c | II-c | III-c | IV-c |

**示例**:
- 心理健康 App = I-a(最低,备案)
- 慢病管理 = I-b(二类,需要 NMPA 注册)
- AI 影像辅助 = III-b(三类,严格审批)
- AI 自动诊断 = III-c(三类,最严)

## 6. SaMD 技术架构特殊性

### 6.1 数据流(必须考虑)

```
┌─────────────────────────────────────────────┐
│         患者端 App(Web H5)                    │
└────────────────┬────────────────────────────┘
                 │ HTTPS / TLS 1.3
                 ▼
┌─────────────────────────────────────────────┐
│      API Gateway + 审计日志                  │
│      (PHI 字段级加密 + 访问控制)              │
└────────────────┬────────────────────────────┘
                 │
       ┌─────────┼─────────┐
       ▼         ▼         ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 用户服务  │ │ 诊疗服务  │ │ 数据服务  │
│ (注册/资料)│ │ (问诊/随访)│ │ (FHIR/AI) │
└────┬─────┘ └────┬─────┘ └────┬─────┘
     │            │            │
     ▼            ▼            ▼
┌──────────────────────────────────┐
│   数据存储(PHI 加密 + 备份)      │
│   - 患者档案 30 年              │
│   - 诊疗记录 30 年              │
│   - 操作日志 6 年              │
└──────────────────────────────────┘
                 │
                 ▼ (HL7 FHIR)
┌──────────────────────────────────┐
│   院内系统集成                  │
│   - HIS / EMR                  │
│   - PACS(影像)                 │
│   - LIS(检验)                  │
└──────────────────────────────────┘
```

### 6.2 关键合规要求(架构层)

| 要求 | 实现 |
|---|---|
| **PHI 加密** | 字段级 AES-256 + 传输 TLS 1.3 |
| **访问控制** | RBAC + 最小权限 + 双因素认证 |
| **审计日志** | 6 年保留 + 不可篡改(WORM 存储) |
| **数据本地化** | 中国数据不出境(重要数据安全评估) |
| **脱敏显示** | 患者/医师/护士 看到不同字段 |
| **同意管理** | 电子知情同意 + 可撤回 |
| **数据可携** | 支持用户导出(参考 GDPR 第 20 条) |
| **灾难恢复** | RTO < 4 小时,RPO < 1 小时 |

## 8. SaMD V5.0 风险与缓解(纯软件版)

| 风险 | 等级 | 缓解 |
|---|---|---|
| **患者数据泄露** | 致命 | 字段级加密 + 审计 + 入侵检测 |

| **医师资质不全** | 高 | 互联网医院牌照 + 医师执业注册 |
| **知识产权** | 中 | 临床路径专利 + 商标 |
| **AI 算法偏见** | 中 | 持续验证 + 多元化数据 |
| **用户误用** | 中 | 明确免责 + 风险提示 |

## 10. 纯软件医疗 vs 硬件医疗对比

| 维度 | 纯软件(SaMD) | 硬件医疗器械 |
|---|---|---|
| **监管核心** | 软件生命周期 + AI/ML | 硬件安全 + 性能 |
| **核心法规** | IMDRF SaMD / FDA Pre-Cert | 传统 510(k)/PMA |
| **IEC 标准** | IEC 62304(软件) | IEC 60601(电气) |
| **临床试验** | 较少(通常) | 必须(通常) |
| **注册周期** | 3-9 个月 | 12-36 个月 |
| **更新方式** | OTA / 软件升级 | 硬件迭代 |
| **网络安全** | **必查** | 部分需要 |
| **上市后监测** | 真实世界数据(RWD) | 不良事件报告 |
| **成本** | 中等 | 高 |

## 12. 临床流程参考(医疗软件必须理解)

### 12.1 在线问诊流程

```
患者主诉(文字/语音/视频)
   ↓
AI 预问诊(可选,辅助分诊)
   ↓
匹配医师(根据科室/专长/评分)
   ↓
在线问诊(视频/图文)
   ↓
电子处方(医师签名,需互联网医院资质)
   ↓
处方审核(药师审核,如需要)
   ↓
药品配送(合作药店)
   ↓
随访/复诊提醒
   ↓
病历归档(30 年)
```

### 12.2 慢病管理流程

```
患者注册(实名认证)
   ↓
健康数据采集(血压/血糖/心率/体重)
   ↓
风险评估(模型预测)
   ↓
个性化干预(饮食/运动/用药提醒)
   ↓
医师介入(异常时)
   ↓
健康报告(月度)
   ↓
复诊提醒
   ↓
长期跟踪(终身)
```
