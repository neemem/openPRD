# 协作流程与版本管理(V1-V9)

> **来源**:从主 SKILL.md 中拆分(主文件保留 1 段话总述 + 链接)。
> 本文件涵盖 V1-V9 全部协作规范,涉及多人/多组协作的项目时查阅。

## 协作总原则

> **单一来源**(Single Source of Truth)+ **明确契约**(Clear Contracts)+ **可追溯**(Traceability)

## V1:文档版本管理

### 版本号规范

```
主版本.次版本.段号-日期
例:v3.0h-20260115
```

- **主版本**:重大重构(≥ 30% 内容变更)
- **次版本**:新增章节或重大调整
- **段号**:对应 06a-06h 子段
- **日期**:YYYYMMDD

### 文档头规范

```markdown
| 版本 | 日期 | 作者 | 说明 |
|------|------|------|------|
| 3.0h | 2026-01-15 | [Your Name] | 段 8:发布管理 + 附录 |
```

### 变更记录

```markdown
## 变更历史

| 版本 | 日期 | 变更人 | 变更内容 |
|------|------|--------|----------|
| 3.0h | 2026-01-15 | 张三 | 初始发布 |
| 3.1h | 2026-02-01 | 李四 | 补充容量规划章节 |
```

## V4:评审流程

```
PM 起草 → 内部自检 → 跨组评审 → 修订 → 终稿 → 归档
   │         │           │         │       │       │
   1d        1d         2-3d      1-2d    1d     永久
```

**评审 Checklist**:
- [ ] 业务规则明确
- [ ] 与上下游文档无冲突
- [ ] 字段/术语一致
- [ ] 可实现性已确认
- [ ] 测试覆盖已规划
- [ ] 风险已识别

## V5:变更管理

### 变更类型

| 类型 | 例子 | 流程 |
|------|------|------|
| **紧急变更** | P0 故障修复 | 即时 + 事后评审 |
| **小变更** | 文案、错别字 | PR + 1 人 review |
| **中等变更** | 新增字段、调整规则 | PR + 2 人 review + 业务确认 |
| **重大变更** | 架构调整、需求变更 | 评审会 + 全员通知 |

### 变更 PR 模板

```markdown
## 变更说明
- 变更类型:[紧急/小/中/大]
- 关联需求:TASK-1234
- 变更原因:

## 变更内容
- [ ] 文档 A: 章节 X 修改
- [ ] 文档 B: 新增章节 Y

## 影响评估
- [ ] 数据库:无 / 有(具体说明)
- [ ] 接口:无 / 有(具体说明)
- [ ] 前端:无 / 有(具体说明)

## 测试
- [ ] 已更新相关测试用例

## 评审
- [ ] PM 评审
- [ ] 技术评审
- [ ] 业务评审(如适用)
```
