# 交付清单（详细参考）

## 概述

文档产出后的交付物清单。本文件与 SKILL.md 保持同步。

---

## 1. 文档产出顺序

```
【第一组】基于调研结果（并行）
├── README.md
├── 02-项目整体说明.md
└── 03-接口文档.md

【第二组】基于第一组结果（并行）
├── 04-前端开发指南.md
├── 05-任务拆分与交付.md
└── 08-内部交互链路.md

【第三组】必须产出
├── 06-产品需求文档.md
└── FigmaMake-Prompt.md
```

---

## 2. 输出目录规范

所有产出文件必须输出到用户当前工作目录下的 `{需求名称}-docs` 目录：

```
{当前工作目录}/
└── {需求名称}-docs/
    ├── README.md
    ├── 02-项目整体说明.md
    ├── 03-接口文档.md
    ├── 04-前端开发指南.md
    ├── 05-任务拆分与交付.md
    ├── 06-产品需求文档.md
    ├── 08-内部交互链路.md
    ├── FigmaMake-Prompt.md
    └── 用户调研报告.md
```

## 3. 最终检查清单

### 3.1 第一组文档（必须）

- [ ] README.md 已生成 → 项目入口
- [ ] 02-项目整体说明.md 已生成 → 背景架构
- [ ] 03-接口文档.md 已生成 → API定义

### 3.2 第二组文档（必须）

- [ ] 04-前端开发指南.md 已生成 → 前端指南
- [ ] 05-任务拆分与交付.md 已生成 → 任务分解
- [ ] 08-内部交互链路.md 已生成 → 交互流程

### 3.3 第三组文档（必须）

- [ ] 06-产品需求文档.md 已生成
- [ ] FigmaMake-Prompt.md 已生成

### 3.4 代码风格检查

- [ ] 颜色值已验证 GlobalStyled.jsx（非近似值）
- [ ] 布局值已验证 LayoutStyled.jsx
- [ ] 使用 Ant Design 组件（非原生HTML）

### 3.5 Mock 数据检查

- [ ] Mock 数据类型已与用户确认
- [ ] 数据规模符合业务场景
- [ ] 敏感数据已脱敏
- [ ] 不确定项已标注"待确认"

### 3.6 关键确认节点

- [ ] 架构设计后 → 用户确认系统架构
- [ ] 方案设计后 → 用户确认技术方案
- [ ] 任务拆分后 → 用户确认任务分解
