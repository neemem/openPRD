# 18 · 数据看板 Checklist(7 条原则落地手册)

> **Why this exists**: 子技能 `openPRD-data-dashboard` 编码了 7 条原则的"流程层"逻辑(Step D1-D7),但每条原则的**细粒度 checklist**、**字段模板**、**6 个 layout 骨架**需要单独的详细参考,避免子技能 SKILL.md 过度膨胀。
>
> **引用方**: `openPRD-data-dashboard/SKILL.md`(主)+ 06c 模板(数据看板特殊页)+ 06d 组件库(KPI 卡片规格)+ 主 SKILL.md 的"是否含数据看板"判断。

---

## §1 原则 1 Checklist(业务背景)

| 检查项 | 通过标准 | 验证方法 |
|--------|----------|----------|
| 业务背景已填 | 1-2 句话,含时间范围 | grep `dashboard-self-check.md` "业务背景" |
| 目标已填 | 引用 02-项目整体说明的 OKR 编号 | grep `OKR-` |
| 用户画像已填 | ≥ 1 个 Persona,含"看数据"动作 | grep `Persona` |
| 看板需求已填 | ≥ 3 条具体需求(不是"看数据"这种空话) | grep `需求` 计数 |

---

## §2 原则 2 Checklist(3-30-300)

每张图必填 1 个层级标签:

| 层级 | 占比建议 | 图表类型 |
|------|----------|----------|
| 3 秒 | 30-40% | 大数字 KPI / 状态色 / 关键趋势箭头 |
| 30 秒 | 40-50% | 折线主图 / Top 列表 / 占比饼 |
| 300 秒 | 10-20% | 多维表 / 明细 / 方法说明 |

**门控**:
- ☐ 没有任何图是"全部 3 层级"(违反 3-30-300)
- ☐ 没有任何图"无层级标签"
- ☐ 3s 区 ≥ 30%,300s 区 ≤ 20%

---

## §3 原则 3 Checklist(总-分-总 + 三句话)

每个中段模块必填 3 句话:

```markdown
### 模块 N: [模块名]
- **一句话数据解读**: [本期 X,环比 +5%,达成率 92%]
- **一句话洞察**: [3 个类目占 80% 增长,新类目增速放缓]
- **一句话行动指引**: [重点资源向 3 个头部类目倾斜,新类目做拉新测试]
```

**门控**:
- ☐ 中段每个模块三句话齐全
- ☐ 三句话必须含**具体数字**或**具体对象**(不能"有所提升"这种空话)
- ☐ "行动指引"必须含**动作动词**和**明确对象**

---

## §4 原则 4 Checklist(管理层四问)

行动表 4 列必齐:

| 列 | 内容要求 | 示例 |
|----|----------|------|
| 谁 | 责任人(到人,不是"运营团队") | 张三(运营经理) |
| 什么时候 | 截止日期(YYYY-MM-DD) | 2026-06-15 |
| 做什么 | 1 句话动作 | 完成 3 个头部类目资源倾斜方案 |
| 预期结果 | 可量化的预期 | GMV 环比提升 8% |

**门控**:
- ☐ 行动表 4 列齐全
- ☐ "谁"必须到人
- ☐ "预期结果"必须可量化

## §5 原则 5 Checklist(首屏 5 件套)

### 5.1 6 个 KPI 卡片字段模板

```markdown
| 指标名 | 当前值 | 同比 | 环比 | 目标差 | 状态 |
|--------|--------|------|------|--------|------|
| GMV | ¥1.2M | +12% | +5% | -3% | 🟡 接近 |
| DAU | 50K | +8% | +2% | +0% | 🟢 达标 |
| 转化率 | 3.2% | -0.5pp | +0.1pp | -0.8pp | 🔴 风险 |
| ... |
```

**KPI 选取合规性**:
- ☐ 数量正好 6(不多不少)
- ☐ 3 个"结果指标" + 2 个"过程指标" + 1 个"健康指标"
- ☐ 状态标签(🟢/🟡/🔴)与目标差正负一致

### 5.2 Hero 标题模板

```
[主题:如"GMV 周报"] · [对象:如"全平台"] · [时间范围:如"2026 W22"] · [阅读价值:如"看 GMV 趋势 + 异常告警"]
```

### 5.3 3 条重点洞察句子模板

```markdown
- 📊 [洞察 1]:本期 GMV ¥1.2M,环比 +5%,主要由 3 个头部类目贡献(占比 80%)
- ⚠️ [洞察 2]:新用户转化率从 3.5% 降至 3.2%,主要受 X 渠道影响
- ✅ [洞察 3]:复购率达 28%,超目标 3pp,Q2 OKR 进度良好
```

## §6 原则 6 Checklist(中段 4 区)

### 4 区固定结构

```markdown
## 趋势主图区(变化)
[多线折线 / 堆叠面积]

## 归因/拆解区(原因)
[瀑布图 / 维度下钻表 / 漏斗]

## 风险与机会区(注意力)
[红绿灯矩阵 / 异常点列表]

## 行动表(谁做什么)
| 谁 | 什么时候 | 做什么 | 预期结果 |
|----|----------|--------|----------|
| ... | ... | ... | ... |
```

**顺序门控**: 必须按"变化 → 原因 → 注意力 → 行动",颠倒即不达标。

## §7 原则 7 Checklist(6 段页面顺序)

```markdown
# 看板标题

## 1. 总览(首屏)         — 一眼看全
## 2. 核心战场(趋势)     — 主战场在哪
## 3. 原因拆解(归因)     — 为什么会这样
## 4. 风险与异常(风险)   — 哪里出问题
## 5. 动作闭环(行动表)   — 该做什么
## 6. 附录追溯(底部)     — 数据源 + 口径 + 责任人
```

**附录追溯段必含**:
- 数据源(哪个系统、哪个库、哪张表)
- 计算口径(如"GMV = 实际支付订单金额 - 退款")
- 责任人(看板维护人,联系信息)
- 更新频率(实时 / T+1 / 周)

## §9 跨文档字段一致性(必查)

按 `reference/15-five-field-crosscheck.md` 规则,数据看板必查字段:

| 字段 | 出现位置 | 必须一致 |
|------|----------|----------|
| `KPI 名` | 06-PRD §5 KPI / 12-数据库 / 11-Mock | 4 处拼写一致 |
| `指标定义` | 12-数据库 字段注释 / 11-Mock 示例 / 06-PRD §5 | 3 处口径一致 |
| `时间范围` | 06-PRD §1 / 06c 页面树 / 附录追溯段 | 3 处时间窗口一致 |

**门控**: 任何字段不一致都会让看板"看着对、用着错"。
