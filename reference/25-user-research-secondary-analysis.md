# 25 · 用户调研二次分析方法论(V5.0 关键)

> 📖 **本文档定位**:
> - 当用户提供 PRD 时,主 Agent 不能"被动接受",必须做**二次深度分析**
> - 当用户"无脑丢"时,主 Agent 必须**主动发现问题 + 提出问题**
> - 这是 V5.0 的关键方法论,不是修改 openPRD Skill,而是为 V5.0 增强做准备

---

## 0. 文档目的

**核心问题**:
- 用户提供的 PRD 可能是"无脑丢"的,他自己也不明白确切内容
- 仅基于 PRD 字面产出,可能**遗漏关键赛道选择、范围问题、目标不一致**
- V4.4 默认直接基于 PRD 产出,缺少"二次分析"环节

**V5.0 改进**:
- 引入**用户调研二次分析方法论**
- 主 Agent 必须执行"超出 PRD"的主动分析
- 必须提出 **10+ 关键问题**,确保调研完整
- 目标:提高执行度 + 置信度 + 目标/范围一致性

---

## 1. 二次分析的 3 个层次

### 1.1 层次 1:PRD 字面理解(Literal)

**内容**:阅读 PRD,理解字面意思

**输出**:文档框架 + 关键模块列表

**V4.4 状态**:✅ 已实施

### 1.2 层次 2:赛道选择分析(Track Selection)

**内容**:PRD 字面之外的**赛道选择和定位**

**关键问题**:
- PRD 描述的产品属于哪个赛道?规模如何?
- 行业内有哪些**直接竞品和间接竞品**?
- 用户的**赛道定位**与头部玩家的差异?
- 是否有**模式借鉴**的对象?

**V5.0 必须执行**:
- 在主 Agent 完成 PRD 字面理解后,**额外启动赛道分析**
- 输出:**赛道选择报告**(融入 14-竞品分析报告 + 14-标杆研究)

**示例**(新兰界项目):
- PRD:微信公众号 H5 兰花行情订阅
- 二次分析:
  - 兰花行业市场规模 ~200 亿
  - 头部玩家:兰花交易网/中国兰花交易网
  - 借鉴对象:**贝壳找房**(真实成交价)+ **我的钢铁网**(会员订阅)
  - 差异定位:垂直行业(兰花)数据 SaaS

### 1.3 层次 3:目标与范围一致性审查(Alignment)

**内容**:**验证 PRD 的目标 / 范围 / 资源 / 时间 是否一致**

**关键问题**:
- 商业模式(订阅)是否与目标用户(45-70 岁)的支付习惯匹配?
- 功能范围(8 大模块)是否与团队规模(7.5 人)在 90 天 MVP 内可达成?
- 性能目标(500 QPS)是否与初期 1 万用户匹配?
- 合规要求(PIPL/EDI)是否已识别?

**V5.0 必须执行**:
- 主 Agent 完成层次 1+2 后,启动**一致性审查**
- 输出:**目标-范围-资源-时间(SCQA)一致性报告**

## 2. 主 Agent 必须提出的 10+ 关键问题

**V5.0 改进**:即使用户给了"完整 PRD",主 Agent 仍必须主动问 **10+ 个关键问题**,确保不遗漏。

### 2.1 赛道选择类(3 题)

| # | 问题 | 目的 |
|---|---|---|
| Q1 | 你认为这个产品属于哪个赛道?为什么? | 验证赛道认知 |
| Q2 | 你的核心差异化定位是什么? | 避免同质化 |
| Q3 | 你希望借鉴哪个**模式**?(贝壳/我的钢铁网/同花顺/其他) | 模式参考 |

### 2.2 目标与范围类(3 题)

| # | 问题 | 目的 |
|---|---|---|
| Q4 | 3 个月 MVP,你的**最关键**3 个功能是什么? | 范围聚焦 |
| Q5 | 你接受哪些功能 **V2.0 推迟**? | 范围取舍 |
| Q6 | 如果只能保留 1 个功能,你会保留哪个? | 核心识别 |

### 2.3 资源与时间类(2 题)

| # | 问题 | 目的 |
|---|---|---|
| Q7 | 你的**实际**团队规模 + 关键成员能力? | 资源对齐 |
| Q8 | 你的**真实**启动时间 + 上线时间节点? | 时间对齐 |

### 2.4 商业模式与合规类(2 题)

| # | 问题 | 目的 |
|---|---|---|
| Q9 | 你的**收入来源**主要是什么?订阅?广告?数据? | 商业模式验证 |
| Q10 | 你的**合规风险**识别?(PIPL/EDI/微信/支付/广告法) | 合规对齐 |

### 2.5 用户与场景类(2+ 题)

| # | 问题 | 目的 |
|---|---|---|
| Q11 | 你的**目标用户**核心 3 类是?各自占比? | 用户聚焦 |
| Q12 | 你的**关键使用场景**前 3 个是? | 场景聚焦 |
| Q13 | 你的**付费转化率**预期是多少? | 商业指标 |

**总计 13 题**,必问。

### 2.6 额外场景问题(根据 PRD 内容动态)

- 如果 PRD 提到"AI":问技术成熟度/数据来源
- 如果 PRD 提到"国际化":问地区/合规/支付
- 如果 PRD 提到"B2B":问销售模式/客户画像
- 如果 PRD 提到"硬件/IoT":问供应链/成本
- ...

## 3. 二次分析的工作流

```
Phase 0: 用户输入 PRD
    ↓
Phase 1: PRD 字面理解(主 Agent)
    - 提取关键模块
    - 提取角色
    - 提取核心场景
    ↓
Phase 2: ⭐ 二次赛道分析(主 Agent 主动)
    - 自动识别赛道
    - 启动 14-竞品分析(已有)
    - 启动 14-标杆研究(已有)
    - 输出赛道定位 + 模式借鉴对象
    ↓
Phase 3: ⭐ 主动提出 10+ 关键问题(主 Agent)
    - 13 题必问(赛道/范围/资源/时间/商业/合规/用户)
    - 根据 PRD 动态追加
    - **用户即使给了"完整 PRD"也必须回答**
    ↓
Phase 4: 用户回答 → 主 Agent 汇总决策矩阵
    ↓
Phase 5: ⭐ 目标-范围-资源-时间一致性审查
    - 检查决策矩阵的一致性
    - 识别**风险点**(如目标 vs 资源不匹配)
    - 提示用户调整
    ↓
Phase 6: 继续后续 Step 1-5
```

## 5. 用户"无脑丢 PRD"场景处理

### 5.1 检测信号

主 Agent 应能识别"用户未深度思考"的信号:

| 信号 | 表现 |
|---|---|
| **PRD 太短** | < 1 页,信息密度低 |
| **缺少量化** | 没有"X 个用户/Y 千万规模/Z 千万营收" |
| **没有竞品分析** | "市面上没有竞品" / "竞品是 XX/XX/XX" |
| **没有商业模式** | "怎么赚钱不知道" |
| **没有优先级** | 所有功能"都很重要" |
| **自相矛盾** | "目标 1000 万用户" + "团队 3 人" |

### 5.2 处理策略

**对"无脑丢"用户,主 Agent 必须**:

1. **明确告知**:"PRD 信息密度低,需要补充 13 个关键问题"
2. **解释重要性**:"不回答这些问题,可能产出偏离方向"
3. **分批提问**:3-4 题/批,避免一次问 13 题
4. **提供推荐选项**:每题带"推荐"选项,降低决策成本
5. **拒绝强行产出**:"信息不足,无法保证置信度"

### 5.3 最低决策门槛

如果用户**拒绝回答 13 题**,主 Agent 应:

```markdown
⚠️ 13 题必问问题用户未全部回答。

V5.0 最低要求:
- 至少 7 题(50%):赛道 + 范围 + 资源
- 推荐 10 题(75%):+ 时间 + 商业
- 理想 13 题(100%):+ 用户场景

如果用户只回答部分:
- V5.0 标记为"低置信度"(< 80 分)
- 产出"假设清单"(标"推断")
- 提示"建议补充回答后重跑"
```

## 6. 二次分析与现有 35 题调研的关系

### 6.1 35 题调研是"事后补"还是"事前问"?

**V4.4 现状**:35 题在 PRD 输入**之后**作为"补充调研"

**V5.0 改进**:35 题 = **二次分析 + 35 题具体问题**
- 二次分析:**主 Agent 自动完成**(赛道 + 一致性)
- 35 题:**必须用户回答**(PRD 信息补全)

### 6.2 35 题与 13 题必问关系

**13 题必问**:**V5.0 强制**(必须问,不可跳过)
**35 题 = 13 必问 + 22 详细问题**(根据项目复杂度可选)

| 用户投入 | 产出质量 | V5.0 处理 |
|---|---|---|
| 回答 13 必问 | 置信度 B 级(80-89) | ✅ 完整产出 |
| 回答 35 题 | 置信度 A 级(90-100) | ✅ 优秀产出 |
| 回答部分 | 置信度 C 级(70-79) | ⚠️ 触发自动扩充 + 假设标注 |
| 拒绝回答 | 置信度 <60 | ❌ 拒绝产出,要求补充 |

## 8. 总结

**V5.0 用户调研二次分析核心理念**:

1. **用户可能"无脑丢"** — 主 Agent 必须主动发现
2. **必须问 10+ 关键问题** — 即使 PRD "看似完整"
3. **二次分析 3 层次** — 字面理解 + 赛道分析 + 一致性审查
4. **置信度量化** — 13 题回答程度直接决定产出质量等级
5. **不能动 Skill 本体** — 所有方法论在 reference/ 文档中

**关键原则**:
- ✅ 用户输入 PRD 也要做二次分析
- ✅ 必问 13 题
- ✅ 赛道选择 + 一致性审查
- ✅ 更高执行度 + 更高置信度
- ✅ 确保目标和范围一致

**openPRD Skill 本体**:✅ 完全不动
**本次新增**:1 个 reference 文件(reference/25)
