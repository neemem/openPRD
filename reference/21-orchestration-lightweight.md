# 21 · 轻量模式调度(3 路行业研究)

> 📖 **使用说明**:Step 0 选"1-3 天 demo"或单角色项目时,5 路行业研究降级为 3 路(行业/合规/技术)。竞品 + 标杆研究跳过,数据源减少,1 小时可完成。本文档是**轻量模式的唯一规则来源**。
>
> **Why**:5 路并行对"内部工具/纯后端 API/快速 demo"是浪费。1 天 demo 跑 5 路 = 4 小时调研,实际项目可能就 4 小时总时间。轻量模式保留核心 3 路,质量不减,时间降到 1/2。

---

## 0. 文档目的

- **触发条件**:什么场景用轻量模式(自动 vs 手动)
- **3 路 vs 5 路**:哪些路跑、哪些不跑
- **降级映射**:5 路中的 2 路(竞品/标杆)在轻量模式下如何用替代方式
- **产出差异**:轻量模式 vs 5 路的最终文档差异

---

## 1. 触发条件

### 1.1 自动触发(主 Agent 自动选)

| 场景 | 时间预算 | 行业研究路数 |
|------|----------|--------------|
| "下周 demo / 1-3 天" | 1-3 天 demo | **3 路轻量** |
| 单角色项目(只选 1 类受众) | 任意 | **3 路轻量** |
| 二期项目(已有产品加功能) | 任意 | **3 路轻量** |
| 内部工具 / 中后台 | 任意 | **3 路轻量** |
| 纯后端 API 服务 | 任意 | **3 路轻量** |

### 1.2 手动触发(用户在 Step 0 问题 D 选)

用户在 Step 0 问题 D 选 "1-3 天 demo" → 主 Agent 默认 3 路轻量。

### 1.3 不降级场景(强制 5 路)

| 场景 | 原因 |
|------|------|
| 商业项目 + 强合规 + 跨境数据 | 5 路全跑保证合规完整 |
| 用户明确说"我要深度研究" | 尊重用户意愿 |
| 1 月+ 长期规划 | 长期项目值得深度调研 |

## 2. 3 路轻量:跑什么 / 不跑什么

| 路 | 标准 5 路 | 轻量 3 路 | 替代方案 |
|----|-----------|-----------|----------|
| **A 行业分析** | ✅ 9 章节 + 波特五力 + PEST | ✅ 保留(简化为 4 章节:行业概况/术语表/3 个核心趋势/Top3 玩家) | 跳过:行业规模测算、波特五力完整版 |
| **B 竞品分析** | ✅ 11 章节 + Battle Card | ⚠️ **跳过主路**,若项目含竞品由 Step 1b Q15-Q18 调研时补充 | 主 Agent 在 35 题 Q15-Q18 主动加问"Top3 竞品" |
| **C 标杆研究** | ✅ 6 步法 + 15+ 来源 | ❌ **完全跳过** | 用 openPRD-deep-research 子技能可手动触发 |
| **D 法规合规** | ✅ 8 章节 + 30+ Checklist | ✅ 保留(简化为 4 章节:必查法规/5 类合规/Top3 风险/应对策略) | 跳过:30+ 完整 Checklist |
| **E 技术趋势** | ✅ 7 大领域 + 成熟度矩阵 | ✅ 保留(简化为 3 领域:核心选型/备选方案/选型评分) | 跳过:7 领域全跑 |

## 3. 降级映射:5 路 → 3 路

```
[标准 5 路]                         [轻量 3 路]
├─ A 行业: 9 章节                   ├─ A 行业: 4 章节(简版)
├─ B 竞品: 11 章节 + Battle Card    ├─ (B 竞品并入 35 题 Q15-Q18)
├─ C 标杆: 6 步法 + 15 来源         ├─ (C 标杆完全跳过)
├─ D 合规: 8 章节 + 30 Checklist    └─ D 合规: 4 章节(简版)
└─ E 技术: 7 领域 + 矩阵                E 技术: 3 领域(简版)
```

## 4. 时间预算

| 模式 | A 行业 | B 竞品 | C 标杆 | D 合规 | E 技术 | 总时间 |
|------|--------|--------|--------|--------|--------|--------|
| **标准 5 路** | 45min | 45min | 60min | 30min | 30min | ~3.5h(并行) |
| **轻量 3 路** | 20min | 跳过 | 跳过 | 15min | 15min | ~50min(并行) |
| **极致 1 路** | 20min | 跳过 | 跳过 | 跳过 | 跳过 | ~20min(应急) |

> **注**:subagent 并行,A+D+E 三路同时跑,总时长 ≈ 最慢一路(~20min)。

## 5. 产出差异

### 5.1 标准 5 路产出

- `14-行业分析报告.md`(~15KB,9 章节)
- `14-竞品分析报告.md`(~12KB,11 章节 + Battle Card)
- `14-标杆研究报告.md`(~10KB,6 步法)
- `14-合规清单.md`(~8KB,8 章节 + 30 Checklist)
- `14-技术趋势报告.md`(~8KB,7 领域)

### 5.2 轻量 3 路产出

- `14-行业分析报告.md`(~5KB,4 章节)
- `14-合规清单.md`(~3KB,4 章节)
- `14-技术趋势报告.md`(~3KB,3 领域)
- (B 竞品 + C 标杆不单独产出,若用户需要可手动触发)

**总产出文档量**:5 路 ~53KB → 3 路 ~11KB(-79%),但核心 3 领域不缩水。

## 6. 主 Agent 决策表

| 输入 | 推荐模式 | 理由 |
|------|----------|------|
| 用户说"下周 demo" | 轻量 3 路 | 时间紧,核心 3 路足够 |
| 用户说"长期项目" | 标准 5 路 | 值得深度调研 |
| 用户说"内部工具" | 轻量 3 路 | 不需要竞品/标杆 |
| 用户说"商业产品 + 海外" | 标准 5 路 | 合规深度必要 |
| 用户说"纯后端 API" | 极致 1 路 | 只需行业概况 |
| Step 0 默认(全选) | 标准 5 路 | 默认全跑,符合 v4.1 老用户习惯 |

## 8. 常见误用(Gotchas)

| Gotcha | 症状 | 修正 |
|--------|------|------|
| L-01: demo 项目跑 5 路 | 时间紧仍全跑,导致 demo 延期 | 1-3 天 demo 强制 3 路 |
| L-02: 轻量模式跳合规 | 项目含金融/医疗仍跳过 D 路 | 强合规项目拒绝轻量 |
| L-03: 轻量模式用 35 题完整版 | demo 项目 50min 调研 | 1-3 天 demo 用 15 题精简 |
| L-04: 轻量模式被滥用 | 老板的项目也被压成 1h | 标准项目识别:商业+合规+海外=标准 5 路 |
| L-05: 轻量模式后用户想加研究 | 用户说"再加个竞品分析" | 主 Agent 可手动触发 B 跑或 C 跑(成本 ~45min) |
