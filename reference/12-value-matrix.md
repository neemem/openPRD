# 16 份文档价值矩阵 + 上下游关系

> **来源**:从主 SKILL.md 中拆分(主文件保留文档清单 + 一句"详见 reference/12")。
> 本文件回答三个问题:**这份文档解决什么**(价值定位)、**回答什么核心问题**、**不回答什么**(边界)。

## 上下游关系总览

```
[调研]
  用户调研报告 ──┐
                ├──→ [设计(产品级)]
  行业分析 ─────┘    项目整体说明 ──┐
                       产品需求文档 ├──→ [开发(技术级)]
                       内部交互链路 │    接口文档 ──┬──→ 前端开发指南
                       FigmaMake ───┘    数据库设计 │     前端交互文档
                                          架构设计  │     Mock 数据文档
                                          后端开发 ─┘     后端开发指南
                                                          ↓
                                                     [测试]
                                                     测试用例
                                                          ↓
                                                     [管理]
                                                     任务拆分与交付
                                                     项目 README
```

> 上下游关系:**调研 → 设计 → 开发 → 测试 → 管理**,每阶段的输出是下一阶段的输入。

## 价值矩阵(16 行完整表)

| # | 文档 | 阶段 | 价值定位(1 句话) | 回答的核心问题 | 不回答什么 | 价值判定标准 |
|---|------|------|--------------------|----------------|------------|--------------|
| 1 | 用户调研报告 | 调研 | 锁定目标用户与场景 | 为谁做?解决什么问题?平台? | 不回答"行业惯例" | 含 5 个非技术问题 + 6 种官方调研方法(anthropics/user-research) + Persona + 置信度评级 |
| 2 | 14-行业分析报告 | 调研 | 提供行业上下文与带引用的研究 | 行业术语?典型流程?标杆产品?市场数据? | 不回答"本项目功能" | 含术语表 + 流程图 + 标杆对照 + 15+ 来源 + 置信度评级 |
| 3 | 02-项目整体说明 | 设计(产品级) | 给出项目蓝图 | 项目是什么?目标?范围?边界? | 不回答"具体功能细节" | 5W(What/Why/Who/Where/When)齐全 |
| 4 | 06-产品需求文档 | 设计(产品级) | 定义功能与验收 | 功能清单?优先级?用户故事? | 不回答"技术实现" | P0 100% 有验收标准 |
| 5 | 08-内部交互链路 | 设计(产品级) | 描述状态流转 | 前后端状态如何切换?异步如何处理? | 不回答"具体 API 字段" | 含时序图或状态机图 |
| 6 | FigmaMake-Prompt | 设计(产品级) | 指导设计还原 | 如何用 AI 还原设计稿?颜色字号? | 不回答"业务逻辑" | 含颜色(已验证)+ 字体 + 组件清单 |
| 7 | 03-接口文档 | 开发(技术级) | 给出技术契约 | API 路径?字段?错误码? | 不回答"为什么这样设计" | 字段名/类型与 PRD/DB 三方一致 |
| 8 | 12-数据库设计 | 开发(技术级) | 给出存储模型 | 表?字段?索引?外键? | 不回答"接口怎么调" | 字段类型与接口文档完全一致 |
| 9 | 13-架构设计 | 开发(技术级) | 给出技术蓝图 | 技术选型?模块拆分?部署? | 不回答"具体业务逻辑" | 含架构图 + 选型理由 + 部署拓扑 |
| 10 | 04-前端开发指南 | 开发(技术级) | 统一前端规范 | 框架?组件库?目录约定?命名? | 不回答"具体页面实现" | 含目录结构 + 组件使用 + 代码片段 |
| 11 | 09-后端开发指南 | 开发(技术级) | 统一后端规范 | 技术栈?分层?错误处理?日志? | 不回答"具体业务实现" | 含分层图 + 错误码约定 + 中间件清单 |
| 12 | 10-前端交互文档 | 开发(技术级) | 描述前端交互 | 页面流转?组件交互?动效? | 不回答"API 字段" | 含流程图 + 状态机 + 动效参数 |
| 13 | 11-Mock 数据文档 | 开发(技术级) | 提供可粘贴的 Mock | Mock 数据长什么样?字段示例? | 不回答"为什么这样设计" | 数据贴合业务(> 3 条示例/接口) |
| 14 | 07-测试用例 | 测试 | 验证质量与验收 | 怎么测 P0?边界?异常? | 不回答"如何修复 bug" | P0 覆盖 100% + 含 E2E 路径 |
| 15 | 05-任务拆分与交付 | 管理 | 排期与验收闭环 | 谁?做什么?几天?怎么验收? | 不回答"具体技术方案" | 每个任务 1-3 天 + 含验收标准 |
| 16 | 01-README | 管理 | 文档入口与索引 | 有什么文档?怎么读?版本? | 不回答"具体内容" | 索引完整 + 阅读路径清晰 |

## 价值链校验要点

| 校验项 | 上游 → 下游 | 失败后果 |
|--------|-------------|----------|
| 字段一致性 | PRD 字段 → 接口 → DB → Mock | 联调返工 |
| 范围一致性 | 整体说明 → 任务拆分 | 任务缺漏或越界 |
| 验收一致性 | PRD 验收 → 测试用例 → 任务验收 | 验收标准不齐 |
| 设计一致性 | FigmaMake → 前端开发指南 → 前端交互 | 视觉偏离 |
