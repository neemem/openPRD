# Multi-Agent 并行处理机制（详细参考）

## 概述

使用 Multi-Agent 并行处理，提高文档产出效率。

---

## 1. 并行任务分组

**⚠️ 遵循原则**：
1. 第一组基于调研结果并行产出
2. 第二组基于第一组结果并行产出
3. 第三组并行产出（不依赖前两组）
4. 独立任务并行执行，依赖任务按顺序执行

```
【第一组】基于调研结果（并行）
├── Agent-A: 生成 README.md
├── Agent-B: 生成 02-项目整体说明.md
└── Agent-C: 生成 03-接口文档.md

【第二组】基于第一组结果（并行）
├── Agent-D: 生成 04-前端开发指南.md
├── Agent-E: 生成 05-任务拆分与交付.md
└── Agent-F: 生成 08-内部交互链路.md

【第三组】（并行，不依赖前两组）
├── Agent-G: 生成 06-产品需求文档.md
└── Agent-H: 生成 FigmaMake-Prompt.md
```

---

## 2. 并行执行模板

```javascript
// 第一组：基于调研结果，并行产出
Promise.all([
  spawnAgent("README", "基于调研结果生成 README..."),
  spawnAgent("项目说明", "基于调研结果生成项目整体说明..."),
  spawnAgent("接口文档", "基于调研结果生成接口文档...")
]).then(() => {
  // 第二组：基于第一组结果，并行产出
  Promise.all([
    spawnAgent("前端指南", "基于第一组结果生成前端开发指南..."),
    spawnAgent("任务拆分", "基于第一组结果生成任务拆分与交付..."),
    spawnAgent("交互链路", "基于第一组结果生成内部交互链路...")
  ])
})

// 第三组：并行，不依赖前两组
Promise.all([
  spawnAgent("产品需求", "基于调研结果生成产品需求文档..."),
  spawnAgent("FigmaPrompt", "基于设计规范生成FigmaMake提示词...")
])
```

## 3. Agent 任务模板

每个 Agent 执行时需要：
1. 读取调研报告
2. 读取对应模板
3. 基于调研结果填充模板
4. 输出文档到 `{需求名称}-docs/` 目录

### 3.1 README 生成

```
输入：调研报告、项目名称
输出：README.md（项目入口）
```

### 3.2 项目整体说明生成

```
输入：调研报告、项目名称
输出：02-项目整体说明.md（背景架构）
```

### 3.3 接口文档生成

```
输入：调研报告、项目名称
输出：03-接口文档.md（API 定义）
```

### 3.4 前端开发指南生成

```
输入：调研报告、第一组产出文档
输出：04-前端开发指南.md（前端实现）
```

### 3.5 任务拆分生成

```
输入：调研报告、第一组产出文档
输出：05-任务拆分与交付.md（任务分解）
```

### 3.6 内部交互链路生成

```
输入：调研报告、第一组产出文档
输出：08-内部交互链路.md（交互流程）
```

### 3.7 产品需求文档生成

```
输入：调研报告、项目名称
输出：06-产品需求文档.md（PRD）
```

### 3.8 FigmaMake 提示词生成

```
输入：设计规范、颜色规范
输出：FigmaMake-Prompt.md（FigmaMake插件提示词）
```

---

## 4. 执行检查清单

每个 Agent 完成后必须核对：

```
第一组（3个文档）
- [ ] README.md
- [ ] 02-项目整体说明.md
- [ ] 03-接口文档.md

第二组（3个文档）
- [ ] 04-前端开发指南.md
- [ ] 05-任务拆分与交付.md
- [ ] 08-内部交互链路.md

第三组（2个文档）
- [ ] 06-产品需求文档.md
- [ ] FigmaMake-Prompt.md
```

---

## 5. 常见问题

| 问题 | 原因 | 修正 |
|------|------|------|
| 只产出部分文档 | agent 执行不完整 | 对照检查清单补全 |
| 文档内容不符合模板 | 未读取模板 | 确保每个 agent 读取模板 |
| 顺序错误 | 未按组执行 | 重新按组并行执行 |
| 目录不一致 | 输出路径错误 | 统一输出到 `{需求名称}-docs/` |
