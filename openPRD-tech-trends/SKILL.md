# openPRD - 技术趋势研究

## 概述

**目的**：识别 **3-5 年技术演进方向**，为产品架构和技术选型提供前瞻性指引。

**重要性**：
- 技术债是 80% 项目的核心痛点（每 5 年要重构）
- 选错技术栈 = 2-3 年的累积损失
- 忽视 AI 演进 = 失去下一代竞争力

**协作位置**：`openPRD-industry-analysis` 的 5 路并行 subagent 之一（与行业分析、竞品分析、标杆研究、法规合规并行）。

## 7 大技术领域

### 1. AI 与大模型

| 趋势 | 成熟度 | 对产品的影响 |
|------|--------|------------|
| **LLM 通用大模型**（GPT-4/Claude 3/Gemini）| 成熟 | 智能客服、文档生成、代码辅助 |

| **RAG（检索增强生成）**| 成熟 | 企业知识库、智能问答 |

## Gotchas (避坑指南)

### G-01: Gartner Hype Cycle 位置可能差 1-2 年,必交叉验证
- **症状**: "RAG 处于 Hype Cycle 创新触发期,2-3 年成熟"——但实际 2025-2026 已大规模商用,不在"创新触发期"
- **原因**: Gartner 报告发布频率低(年度),实际技术演进快于报告
- **修正**: Hype Cycle 必 3 路交叉验证:(1) **Gartner / Forrester** 报告(2) **vendor 实际定价**(无定价 = 未商用)(3) **GitHub / npm 下载量趋势**(单月 +30% = 爆发期)。3 路不一致必标"分歧"+"团队判断"。
- **参考**: openPRD-tech-trends §1-7 技术领域

### G-02: 选"领导公司在用的技术"而不管是否合用
- **症状**: 看到 Salesforce 用"自己的 LLM + 向量数据库"就推"我们也用 LLM + Milvus"——但 10 人团队根本维护不起
- **原因**: 误把"标杆技术栈"当"标准答案",忽略团队规模/能力/预算
- **修正**: 技术选型必含 3 字段:**为什么选 / 团队匹配度(规模 + 技能)/ 风险与备选**。标杆方案 = "上限参考",不是"现成答案"。例:Salesforce 用 LLM = 大厂方案,小团队应接 GPT API 而非自建。
- **参考**: openPRD-tech-trends §本项目技术选型建议(具体库 + 版本 + 理由 + 风险)

### G-03: AI 章节只写"我们要用大模型",没写"用在哪 / 数据从哪 / 评测怎么做"
- **症状**: 趋势报告写"项目将集成 LLM 实现智能客服",但漏掉 3 个落地问题:数据来源、Prompt 评测、合规(数据出境)
- **原因**: AI 是"必做"的政治正确,agent 凑字数
- **修正**: AI 章节必含 5 段:**应用场景 / 数据来源(自有 / 公开 / 爬取)/ 模型选择(自建 / API)/ 评测指标(准确率 / 幻觉率)/ 合规(PIPL / 跨境)**。5 段齐全才能"建议使用"。见 examples/01 §AI 章节(智云 CRM 智能客服 + 推荐 + 营销 + 数据查询)。
- **参考**: examples/01 §04-tech-trends.md

### G-04: 信创/国产化只在"政府项目"提,实际 B2B 金融/央国企也要
- **症状**: B2B 金融业务报告只字未提"信创适配",但客户招标明确要求"麒麟 OS + 达梦 DB"
- **原因**: agent 默认"信创 = 政府项目",漏 B2B 央国企客户
- **修正**: **B2B 项目必查客户行业**——金融/能源/电信/交通/医疗/教育央国企,都可能在信创清单。信创评估必含 3 字段:**目标 OS / 数据库 / 中间件**(每项标"国产化要求强度")。见 examples/01 §7 国产化(信创)。
- **参考**: examples/01 §7 国产化(信创)

### G-05: 前端趋势报告只提"用 React",没提 React 19 RSC / Server Actions / 新 SSR 范式
- **症状**: 选型写"前端用 React 18",但 2026 年 React 19 已是默认(带 RSC / Server Actions / 新编译器)
- **原因**: agent 训练数据停在 2024,没跟进 React 19
- **修正**: 前端选型必查"上一代 vs 当前代"差异,必调用 openPRD-context7-integration 验证库当前 API 状态(不是查文档章节标题)。技术选型章节必含**版本 + 发布时间 + 关键新特性**。
- **参考**: openPRD-context7-integration §查询质量要点
