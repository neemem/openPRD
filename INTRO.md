# openPRD

> **从一份 PRD 到完整产品文档,AI 自动帮你完成。**

---

## 一句话介绍

**openPRD** 是一个 AI 工作流 Skill:你提供一份产品需求文档(PRD),它通过 35 题用户调研 + 5 路行业研究 + 39 份产品文档,自动产出**可直接用于开发的产品文档包**。

不是文档生成器,是**需求到交付的完整工作流**。

---

## 这个 Skill 适合你吗?

### ✅ 适合(典型用户)

- 📱 **产品经理**:有 PRD,想快速产出完整文档
- 🏗️ **技术负责人**:想统一团队文档规范
- 💼 **创业团队**:人手少,需要 AI 加速
- 📚 **咨询/外包**:同时接多个项目,需要标准化交付

### ❌ 不适合(其他工具更合适)

- 想要 AI 帮你写代码 → 用 Cursor / Copilot
- 已经有完整文档 → 没必要
- 项目极简(< 5 份文档) → 直接写更快
- 强合规项目(医疗/金融) → 需要额外配置

---

## 这个 Skill 能做什么?

### 一句话能力

**输入一份 PRD → 输出 39 份完整产品文档**

### 详细能力

| 能力 | 说明 |
|---|---|
| **35 题用户调研** | AI 自动问 35 个关键问题(分 9 批) |
| **5 路行业研究** | 行业/竞品/标杆/合规/技术 并行调研 |
| **完整 PRD 产出** | 16 章核心 PRD + 9 份拆分文档 |
| **技术架构设计** | 数据库 + 架构 + 接口 + 开发指南 |
| **法律协议** | 4 份(会员/用户/退款/隐私) |
| **运维文档** | 扩容/日志/灾备/监控告警 |
| **质量门控** | 4 级门控 + 100 分制评分 |

### 实际产出(新兰界项目实测)

```
📦 39 份文档
📊 总计 1.81 MB
📏 平均 47 KB / 份
✅ 全部 ≥ 40K(100%)
⭐ 置信度 87/100
⏱️  耗时 ~80 分钟
```

## 使用流程(简单 3 步)

### Step 1:安装(2 分钟)

```bash
# 下载 zip 包
unzip openPRD-v5.0-skill.zip
cp -r openPRD/ ~/.claude/skills/
```

### Step 2:触发(几秒钟)

在 Claude Code CLI 中:
```
/openPRD
```

### Step 3:回答问题(约 25 分钟)

AI 会问 35 个关键问题,分 9 批:
- 你的产品是什么?
- 目标用户是谁?
- 商业模式?
- 竞品?
- 性能要求?
- ...

**完成!** AI 自动产出 39 份文档。

## 这个 Skill 有什么不同?

### 与"AI 文档生成器"对比

| 维度 | 普通 AI 生成 | **openPRD** |
|---|---|---|
| 内容深度 | 通常 5-10K | **40K+ 强制** |
| 结构 | 自由发挥 | **16 章标准 PRD** |
| 行业研究 | 无 | **5 路并行调研** |
| 跨文档一致性 | 容易矛盾 | **5 字段交叉验证** |
| 质量保证 | 凭运气 | **4 级门控 + 评分** |
| 法律合规 | 用户自己加 | **4 份法律协议** |
| 工具产出 | 单个文档 | **完整文档包** |

### 核心特色

- ✅ **方法论驱动**:35 题调研 + 6 步流程,不是随便写
- ✅ **质量门控**:4 级门控 + 100 分制评分,内容不达 80 分不过
- ✅ **实战验证**:新兰界项目 39 份文档全部产出
- ✅ **可定制**:不修改 Skill 本体,只新增 reference/ 文档
- ✅ **可扩展**:支持 V5.1+ 医疗等强合规领域
