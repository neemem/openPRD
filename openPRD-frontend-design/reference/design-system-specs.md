# 设计规范完整输出格式

以下为**必须产出**的完整设计规范模板，**不得简化**。

---

## 3.1 设计方向定位

```markdown
## 本项目设计方向

**设计理念**：[一句话核心定位，如"极简克制的工具美学"]

**视觉调性**：
- 主色调：[色相倾向，如"冷灰蓝传递专业感"]
- 辅以：[点缀色，如"活力橙传递年轻感"]
- 整体氛围：[如"克制但不冰冷"]

**Typography 方向**：
- 标题字体：[推荐方向，如"几何感强的无衬线"]
- 正文字体：[推荐方向，如"清晰易读的柔和无衬线"]

**空间与布局**：
- 密度倾向：[高密度信息展示 / 留白呼吸感]
- 圆角倾向：[锐利直角 / 柔和圆角]
- 卡片风格：[无边框悬浮 / 轻边框分隔 / 纯色块]

**动效期望**：
- 节奏：[快速响应 / 舒缓从容]
- 特色：[如"加载时有骨架屏过渡"]
```

## 3.2 色彩规范

```markdown
## 色彩规范

### 主色（Primary）
- 色值：[精确十六进制]
- 用途：[按钮、链接、强调]
- 无障碍：需确认 WCAG 对比度 ≥ 4.5:1

### 功能色
| 用途 | 色值 | 说明 |
|------|------|------|
| 成功 | #15b79f | 状态标识 |
| 警告 | [色值] | 警示状态 |
| 错误 | #fb5248 | 错误、删除 |
| 信息 | [色值] | 提示信息 |

### 中性色
| 用途 | 色值 |
|------|------|
| 主文字 | #212636 |
| 辅助文字 | #667085 |
| 边框 | #E4E7EB |
| 背景 | #F7F8F9 |

### 禁止使用
- ❌ 纯黑 #000000（文字）
- ❌ 纯白 #FFFFFF（背景，如需深色背景用 #F7F8F9）
- ❌ 渐变色作为主色调
```

## 3.3 Typography 规范

```markdown
## Typography 规范

### 字体选择原则
- 标题：[推荐字体方向，说明理由]
- 正文：[推荐字体方向，说明理由]
- 等宽：[代码/数据展示用]

### 字号阶梯
| 层级 | 字号 | 行高 | 字重 | 用途 |
|------|------|------|------|------|
| H1 | 28px | 36px | 600 | 页面标题 |
| H2 | 22px | 30px | 600 | 区块标题 |
| H3 | 16px | 24px | 600 | 卡片标题 |
| Body | 14px | 22px | 400 | 正文 |
| Small | 12px | 18px | 400 | 辅助文字 |

### 字重使用规范
- 600：标题、重要数字、按钮文字
- 400：正文、说明文字
- 300：辅助信息、占位符
```

- 圆角：8px

## 3.6 动效规范

```markdown
## 动效规范

### 动效原则
- 有意义：动效传递信息（状态变化、层级关系）
- 有克制：不过度动画，不影响效率
- 有一致：相同场景的动效保持一致

### 常用动效
| 类型 | 时长 | 缓动 | 用途 |
|------|------|------|------|
| 微交互 | 150ms | ease-out | hover、focus |
| 展开/收起 | 250ms | ease-in-out | 折叠面板 |
| 页面过渡 | 300ms | ease-out | 路由切换 |
| 加载骨架 | 1.5s | linear | 骨架屏循环 |

### 禁止
- ❌ 超过 500ms 的单次动效
- ❌ 循环动画用于非加载状态
- ❌ 过于花哨的 3D 效果
```

## 3.7 Figma 设计规范

```markdown
## Figma 设计规范

### 设计系统选择
- 使用：[客户已有设计系统 / 新建专用文件]
- 组件库：[建议使用的开源组件库方向]

### 画板设置
- 画板尺寸：根据目标设备
  - PC 端：1440px 宽度（最大）
  - 移动端：375px / 414px
- 网格：8px 网格系统
- 标尺：显示 8px 间距

### 图层命名
- 格式：[类型]-[名称]-[状态]
- 示例：Btn-提交-hover、Input-用户名-focus

### 标注要求
- 所有间距必须标注
- 所有颜色必须标注（十六进制值）
- 字号、字重、行高必须完整标注
```
